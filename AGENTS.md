# AGENTS.md (Repository Root)

## 0) 문서 목적
이 문서는 이 저장소에서 작업하는 에이전트/기여자를 위한 전역 작업 규칙입니다.  
목표는 **일관성, 안전성, 추적 가능성**을 높이는 것입니다.

---

## 1) 적용 범위 및 우선순위
- 적용 범위: 이 파일이 위치한 디렉터리 이하 전체 트리
- 우선순위:
  1. 사용자/시스템의 직접 지시
  2. 더 하위 디렉터리의 `AGENTS.md`
  3. 이 루트 `AGENTS.md`
- 충돌 시: **더 구체적인(더 하위) 규칙이 우선**합니다.

---

## 2) 작업 원칙 (필수)
1. **요청 범위 준수**: 요청된 목표와 무관한 수정 금지
2. **최소 변경**: 필요한 파일만, 필요한 줄만 수정
3. **기존 관례 우선**: 현재 코드 스타일/아키텍처/네이밍 존중
4. **검증 우선**: lint/test/build 가능한 범위에서 반드시 수행
5. **재현 가능성**: 실행 명령, 결과, 실패 사유를 명확히 기록
6. **보안 준수**: 비밀정보 하드코딩/커밋 금지

---

## 3) 표준 작업 절차
1. 현재 상태 파악
   - 디렉터리 구조, 실행/테스트 명령, 관련 설정 확인
2. 변경 계획 수립
   - 어떤 파일을 왜 바꾸는지 간단히 정리
3. 구현
   - 사이드 이펙트 최소화
4. 검증
   - 가능하면 `lint -> unit test -> build` 순으로 수행
5. 결과 보고
   - 변경 요약 / 검증 결과 / 남은 리스크 기록

---

## 4) 코드/문서 스타일 가이드
- 포매터/린터가 있으면 그 규칙을 최우선 적용
- 불필요한 리네임/대규모 포맷 변경 금지
- 주석은 “무엇”보다 “왜” 중심
- 공개 인터페이스 변경 시 영향 범위 명시
- 문서 변경 시 예시/명령은 실행 가능한 형태로 작성

---

## 5) 테스트 및 품질 기준
- 변경 유형별 최소 기준:
  - 문서만 변경: 링크/구문/구조 검토
  - 로직 변경: 관련 단위 테스트 수행
  - 설정 변경: 영향받는 실행 경로 검증
- 테스트 실패 시:
  - 실패 원인
  - 영향 범위
  - 임시 우회책(있다면)
  를 함께 기록

---

## 6) 커밋 규칙
- 한 커밋 = 하나의 논리 단위
- 커밋 메시지 형식(권장):
  - `feat: ...`
  - `fix: ...`
  - `docs: ...`
  - `refactor: ...`
  - `chore: ...`
- 예시:
  - `docs: add root and frontend AGENTS guidelines`
  - `fix: prevent null dereference in auth middleware`

---

## 7) PR 작성 규칙
PR 본문에 아래 항목 필수:
1. 배경(왜 필요한가)
2. 변경 사항(무엇이 달라졌나)
3. 검증 방법/결과(어떻게 확인했나)
4. 영향 범위(어디가 영향 받나)
5. 롤백 방법(문제 시 되돌리는 방법)

---

## 8) 보안/컴플라이언스 체크리스트
- [ ] API 키/토큰/비밀번호 커밋 없음
- [ ] 개인정보/민감정보 로그 노출 없음
- [ ] 외부 의존성 추가 시 신뢰성 확인
- [ ] 라이선스 충돌 가능성 점검

---

## 9) 금지 사항
- 요청되지 않은 대규모 리팩터링
- 검증 없이 “정상 동작” 주장
- 범위 밖 파일 수정
- 임시 디버그 코드/로그 그대로 커밋
