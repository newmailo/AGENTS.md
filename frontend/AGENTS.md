# AGENTS.md (Subdirectory)

> 이 파일은 상위 AGENTS.md를 보완하며, 이 디렉터리 하위에만 적용됩니다.

## 적용 범위
- 현재 디렉터리 및 모든 하위 디렉터리
- 상위 규칙과 충돌 시, 이 파일 규칙이 우선

## 이 영역의 책임
- (예시) 사용자 UI, API 서버, 배치 작업, 인프라 코드 등
- 이 디렉터리의 변경은 해당 책임 범위를 벗어나지 않도록 합니다.

## 로컬 규칙
1. 이 디렉터리의 네이밍/아키텍처 관례를 유지합니다.
2. 공통 모듈 사용 우선순위를 따릅니다. (중복 구현 금지)
3. 공개 인터페이스 변경 시 하위 호환성/마이그레이션 여부를 명시합니다.

## 구현 가이드
- 신규 파일 생성 시 위치 기준:
  - 기능 코드: `...`
  - 테스트 코드: `...`
  - 설정/상수: `...`
- 에러 처리/로깅 규칙:
  - (예시) 사용자 노출 메시지와 내부 로그 분리
- 성능/보안 체크포인트:
  - N+1 조회, 불필요한 렌더링, 입력 검증 누락 등

## 테스트 규칙 (이 영역 전용)
- 최소 실행:
  - `...` (예: unit test command)
- 변경 유형별 추가 테스트:
  - API 스펙 변경 시: contract/integration test
  - UI 변경 시: snapshot/e2e 또는 수동 확인 절차

## 리뷰 포인트
- 리뷰어가 특히 봐야 할 항목:
  1. 비즈니스 로직 정확성
  2. 예외/경계값 처리
  3. 성능/보안 영향
  4. 테스트 커버리지

## 문서화
- 동작/스펙 변경 시 다음 문서 중 해당 항목 업데이트:
  - `README.md`
  - `docs/...`
  - API 스키마/예시

## 금지 사항 (이 영역 전용)
- 이 영역 외부 의존성에 대한 무단 인터페이스 변경
- 테스트 없는 핵심 로직 변경
- 임시 디버그 코드/로그 커밋
